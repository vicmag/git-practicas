Comparación de Notaciones para Stubs en Mockito

Diferencias Clave entre when(), doReturn() y given()

Notación Sintaxis Ventajas Limitaciones Casos de Uso
when() when(mock.metodo()).thenReturn(valor)
  • Sintaxis más natural
  • Fácil de leer
  • Llama realmente al método
  • Problemas con métodos void
 Métodos que devuelven valores y no causan efectos secundarios
doReturn() doReturn(valor).when(mock).metodo()
  • No llama al método real
  • Funciona con métodos void
  • Mayor control
  • Sintaxis menos intuitiva
  • Métodos void
  • Spies
  • Cuando no se debe ejecutar el método real
given() (BDDMockito) given(mock.metodo()).willReturn(valor)
  • Estilo BDD (Given-When-Then)
  • Más legible en contextos BDD
  • Requiere importar BDDMockito
  • Mismos problemas que when()
 Cuando se sigue metodología BDD o se busca mejor legibilidad

Ejemplos Prácticos con el Escenario de Fase Roja

1. Notación when().thenReturn()

@Test
void incrementStock_ValidIncrement_WhenNotation_ShouldUpdateProductStock() {
    // Arrange
    String productName = "Camiseta Azul";
    int initialStock = 10;
    int increment = 5;
    Product product = new Product(productName, initialStock);
    
    ProductService productService = new ProductService(productRepository);
    when(productRepository.findByName(productName)).thenReturn(product);
    
    // Act
    productService.incrementStock(productName, increment);
    
    // Assert
    assertEquals(initialStock + increment, product.getStock());
    verify(productRepository).findByName(productName);
    verify(productRepository).save(product);
}

2. Notación doReturn().when()

@Test
void incrementStock_ValidIncrement_DoReturnNotation_ShouldUpdateProductStock() {
    // Arrange
    String productName = "Camiseta Azul";
    int initialStock = 10;
    int increment = 5;
    Product product = new Product(productName, initialStock);
    
    ProductService productService = new ProductService(productRepository);
    doReturn(product).when(productRepository).findByName(productName);
    
    // Act
    productService.incrementStock(productName, increment);
    
    // Assert
    assertEquals(initialStock + increment, product.getStock());
    verify(productRepository).findByName(productName);
    verify(productRepository).save(product);
}

3. Notación given().willReturn() (BDD)

@Test
void incrementStock_ValidIncrement_GivenNotation_ShouldUpdateProductStock() {
    // Arrange
    String productName = "Camiseta Azul";
    int initialStock = 10;
    int increment = 5;
    Product product = new Product(productName, initialStock);
    
    ProductService productService = new ProductService(productRepository);
    given(productRepository.findByName(productName)).willReturn(product);
    
    // Act
    productService.incrementStock(productName, increment);
    
    // Assert
    assertEquals(initialStock + increment, product.getStock());
    then(productRepository).should().findByName(productName);
    then(productRepository).should().save(product);
}

Recomendaciones de Uso

  • Prefiere doReturn().when() cuando:
    • Trabajes con spies
    • No quieras que se ejecute el método real
    • Configures mocks para métodos void
  • Usa when().thenReturn() para:
    • Código más legible en casos simples
    • Cuando no importa que se ejecute el método real momentáneamente
  • Elige given().willReturn() cuando:
    • Sigas metodología BDD
    • Busques consistencia en el estilo Given-When-Then

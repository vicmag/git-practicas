🛒 Caso de Uso: Problema con la Pasarela de Pago

Escenario

Un usuario intenta realizar una compra en un e-commerce, pero después de ingresar sus datos de pago, la página queda en estado de “cargando” sin completar la transacción. El equipo sospecha que hay un problema en la comunicación entre el frontend y la pasarela de pago.

🔍 Prueba Usando HAR

1️⃣ Captura del Archivo HAR

Para analizar el problema, un tester o desarrollador debe capturar el tráfico de red desde el navegador:

1. Abrir la tienda en línea en Google Chrome o Firefox.

2. Ir a Herramientas para desarrolladores (F12 o Ctrl + Shift + I).

3. Ir a la pestaña Red (Network).

4. Habilitar Preservar Registro (Preserve log) para registrar todas las solicitudes.

5. Realizar la compra normalmente hasta que falle.

6. Guardar el registro como archivo HAR (Export HAR).

2️⃣ Análisis del Archivo HAR

El archivo HAR contiene información clave como:

• Tiempo de carga de cada solicitud.

• Códigos de estado HTTP (200, 400, 500, etc.).

• Cuerpo de las respuestas.

• Headers de la petición.

3️⃣ Creación de una Prueba Automatizada en Java (JUnit + HttpClient)

Para validar que la pasarela de pago está funcionando correctamente, podemos escribir una prueba en Java con JUnit 5 y HttpClient:

import static org.junit.jupiter.api.Assertions.assertEquals;
import java.net.URI;
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;
import org.junit.jupiter.api.Test;

public class PasarelaPagoTest {

    @Test
    void testTransaccionExitosa() throws Exception {
        HttpClient client = HttpClient.newHttpClient();

        String jsonData = """
                {
                    "monto": 100.00,
                    "moneda": "USD",
                    "tarjeta": "4111111111111111",
                    "expiracion": "12/25",
                    "cvv": "123"
                }
                """;

        HttpRequest request = HttpRequest.newBuilder()
                .uri(URI.create("https://api.pagos.com/transaction"))
                .header("Authorization", "Bearer TOKEN_VALIDO")
                .header("Content-Type", "application/json")
                .POST(HttpRequest.BodyPublishers.ofString(jsonData))
                .build();

        HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());

        assertEquals(200, response.statusCode(), "La transacción no fue exitosa.");
    }
}

✅ Beneficios del Uso de HAR en la Prueba

• Diagnóstico Rápido: Se identifica el problema en la autenticación sin revisar logs del servidor.

• Automatización: Se puede validar la integración con la pasarela en pruebas automatizadas.

• Reproducibilidad: El archivo HAR permite compartir el problema con otros equipos.

Este ejemplo demuestra cómo un archivo HAR puede ser una herramienta clave en pruebas de servicios en retail, ayudando a depurar problemas en la integración de pagos en línea.

Tutorial Básico de Pruebas Unitarias

1. Fundamentos de Pruebas Unitarias

Una prueba unitaria verifica el comportamiento de una unidad de código (método, clase) de forma aislada, rápida y repetible.

Ejemplo Básico:

// Clase a probar
public class Calculadora {
    public int sumar(int a, int b) {
        return a + b;
    }
}

// Prueba unitaria
import static org.junit.jupiter.api.Assertions.assertEquals;
import org.junit.jupiter.api.Test;

class CalculadoraTest {
    @Test
    void sumar_DosNumerosPositivos_RetornaSuma() {
        // Arrange (preparar)
        Calculadora calc = new Calculadora();
        
        // Act (actuar)
        int resultado = calc.sumar(3, 5);
        
        // Assert (verificar)
        assertEquals(8, resultado);
    }
}

2. Estructura AAA (Arrange-Act-Assert)

Toda prueba unitaria bien estructurada sigue este patrón:

a) Arrange (Preparación)

Configura el escenario de prueba:

ProductRepository repoMock = mock(ProductRepository.class);
ProductService service = new ProductService(repoMock);
Product producto = new Product("Camiseta", 10);

b) Act (Ejecución)

Ejecuta el método bajo prueba:

service.setMinimumStockLevel(producto, 15);

c) Assert (Verificación)

Comprueba el resultado esperado:

assertEquals(15, producto.getMinimumStockLevel());
verify(repoMock).save(producto);

3. Uso Básico de Mockito

Mockito permite crear objetos simulados para aislar la unidad bajo prueba.

a) Crear un mock:

ProductRepository repoMock = mock(ProductRepository.class);

b) Configurar comportamiento:

when(repoMock.findByName("Camiseta"))
    .thenReturn(new Product("Camiseta", 10));

c) Verificar interacciones:

verify(repoMock).save(any(Product.class));

4. Tipos Comunes de Pruebas

Prueba de camino feliz:

@Test
void setMinimumStockLevel_ValorValido_ActualizaStock() {
    ProductRepository repoMock = mock(ProductRepository.class);
    ProductService service = new ProductService(repoMock);
    Product producto = new Product("Camiseta", 10);
    
    service.setMinimumStockLevel(producto, 15);
    
    assertEquals(15, producto.getMinimumStockLevel());
}

Prueba de validación:

@Test
void setMinimumStockLevel_ValorCero_LanzaExcepcion() {
    ProductRepository repoMock = mock(ProductRepository.class);
    ProductService service = new ProductService(repoMock);
    Product producto = new Product("Camiseta", 10);
    
    assertThrows(IllegalArgumentException.class, () -> {
        service.setMinimumStockLevel(producto, 0);
    });
}

5. Consejos para Buenas Pruebas

  • Nombres descriptivos: metodoPrueba_condicion_esperado
  • Una sola responsabilidad: Cada prueba verifica un solo comportamiento
  • Independencia: Las pruebas no deben depender entre sí
  • Rapidez: Deben ejecutarse en milisegundos
  • Mocks para dependencias: Base de datos, servicios externos, etc.

Tutorial Completo de Pruebas Unitarias con JUnit y Mockito

4. Funcionalidades Avanzadas de Mockito

4.1 Uso de @Spy

Un spy es un objeto real que podemos "espiar" y configurar parcialmente:

@ExtendWith(MockitoExtension.class)
class ProductServiceSpyTest {

    @Spy
    private ProductRepository realRepository = new DatabaseProductRepository();
    
    @InjectMocks
    private ProductService productService;
    
    @Test
    void whenUpdateStock_thenVerifyRealMethodCalled() {
        Product product = new Product("Camiseta", 10);
        
        // Configurar comportamiento específico para el spy
        doReturn(product).when(realRepository).findByName("Camiseta");
        
        productService.updateStock("Camiseta", 5);
        
        // Verificar que se llamó al método real
        verify(realRepository).save(any(Product.class));
    }
}

Diferencia entre @Mock y @Spy:

  • @Mock: Crea un mock completo, todos los métodos están "vacíos" a menos que se configuren
  • @Spy: Usa una instancia real, pero permite anular comportamientos específicos

4.2 Uso de @InjectMocks

@InjectMocks crea una instancia de la clase bajo prueba e inyecta automáticamente los mocks:

@ExtendWith(MockitoExtension.class)
class ProductServiceTest {

    @Mock
    private ProductRepository productRepository;
    
    @Mock
    private StockAlertNotifier notifier;
    
    @InjectMocks
    private ProductService productService; // Inyecta los mocks anteriores
    
    @Test
    void whenProductStockLow_thenNotifyAlert() {
        Product product = new Product("Camiseta", 5, 10);
        when(productRepository.findByName("Camiseta")).thenReturn(product);
        
        productService.checkStock("Camiseta");
        
        verify(notifier).notifyLowStock(product);
    }
}

4.3 Uso de @ExtendWith(MockitoExtension.class)

Esta anotación inicializa automáticamente los mocks y maneja la inyección:

@ExtendWith(MockitoExtension.class) // Habilita Mockito para JUnit 5
class OrderServiceTest {

    @Mock
    private PaymentGateway paymentGateway;
    
    @Mock
    private OrderRepository orderRepository;
    
    @InjectMocks
    private OrderService orderService;
    
    @Test
    void processOrder_WithValidPayment_ShouldSaveOrder() {
        Order order = new Order("123", 100.0);
        when(paymentGateway.process(anyDouble())).thenReturn(true);
        
        orderService.process(order);
        
        verify(orderRepository).save(order);
    }
}

Ventajas:

  • Elimina la necesidad de llamar manualmente a MockitoAnnotations.openMocks(this)
  • Integración más limpia con JUnit 5
  • Inyección automática de dependencias

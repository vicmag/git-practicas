Ejemplos de Violaciones SOLID y sus Métricas Asociadas

1. SRP - Alta Complejidad Ciclomática

Métrica Sonar: Complejidad Ciclomática alta (>10)

Problema: Clase/Método con múltiples responsabilidades

Violación (Código Malo):

public class OrderProcessor {
    // Este método hace demasiado: validación, cálculo, notificación, persistencia
    public void processOrder(Order order) {
        // 1. Validar orden
        if (order == null) throw new IllegalArgumentException();
        if (order.getItems().isEmpty()) throw new IllegalStateException();
        
        // 2. Calcular total
        double total = 0;
        for (Item item : order.getItems()) {
            total += item.getPrice() * item.getQuantity();
            if (item.isTaxable()) {
                total += calculateTax(item);
            }
        }
        
        // 3. Aplicar descuentos
        if (order.getCustomer().isPremium()) {
            total *= 0.9; // 10% descuento
        }
        
        // 4. Guardar en base de datos
        order.setTotal(total);
        orderRepository.save(order);
        
        // 5. Notificar al cliente
        emailService.sendConfirmation(order.getCustomer(), order);
    }
    
    private double calculateTax(Item item) { ... }
}

Solución (Código Bueno):

public class OrderProcessor {
    private final OrderValidator validator;
    private final OrderCalculator calculator;
    private final OrderRepository repository;
    private final NotificationService notifier;
    
    public void processOrder(Order order) {
        validator.validate(order);
        calculator.calculateTotal(order);
        repository.save(order);
        notifier.sendConfirmation(order);
    }
}

// Cada clase maneja una sola responsabilidad

2. OCP - Código Duplicado

Métrica Sonar: Líneas duplicadas

Problema: Modificación directa en lugar de extensión

Violación (Código Malo):

public class DiscountCalculator {
    public double applyDiscount(Order order, String discountType) {
        if ("SUMMER".equals(discountType)) {
            return order.getTotal() * 0.8; // 20% descuento
        } else if ("BLACKFRIDAY".equals(discountType)) {
            return order.getTotal() * 0.7; // 30% descuento
        }
        // Cada nuevo descuento requiere modificar esta clase
        return order.getTotal();
    }
}

Solución (Código Bueno):

public interface DiscountStrategy {
    double apply(Order order);
}

public class SummerDiscount implements DiscountStrategy { ... }
public class BlackFridayDiscount implements DiscountStrategy { ... }

public class DiscountCalculator {
    public double applyDiscount(Order order, DiscountStrategy strategy) {
        return strategy.apply(order);
    }
}
// Nuevos descuentos se añaden con nuevas clases sin modificar existentes

3. LSP - Herencia Profunda

Métrica Sonar: Profundidad de herencia (>3 niveles)

Problema: Jerarquía compleja que viola sustituibilidad

Violación (Código Malo):

class Bird {
    public void fly() { ... }
}

class Penguin extends Bird {
    @Override
    public void fly() {
        throw new UnsupportedOperationException("Penguins can't fly!");
    }
}

// Uso que viola LSP
public void makeBirdFly(Bird bird) {
    bird.fly(); // Falla si es un pingüino
}

Solución (Código Bueno):

interface Bird {
    void move();
}

class FlyingBird implements Bird {
    public void move() { fly(); }
    private void fly() { ... }
}

class Penguin implements Bird {
    public void move() { swim(); }
    private void swim() { ... }
}

// Todas las aves pueden moverse, cada una a su manera

4. ISP - Interfaces Gruesas

Métrica Sonar: Muchos métodos por interfaz (>5)

Problema: Clientes obligados a implementar métodos no usados

Violación (Código Malo):

interface Animal {
    void eat();
    void sleep();
    void fly();   // ¿Y los pingüinos?
    void swim();  // ¿Y los gatos?
    void crawl(); // ¿Y los pájaros?
}

class Bird implements Animal {
    // Debe implementar todos los métodos aunque no los use
    public void crawl() { /* no aplica */ }
}

Solución (Código Bueno):

interface BasicAnimal {
    void eat();
    void sleep();
}

interface FlyingCreature {
    void fly();
}

interface SwimmingCreature {
    void swim();
}

class Bird implements BasicAnimal, FlyingCreature {
    // Solo implementa lo que necesita
    public void fly() { ... }
}

5. DIP - Alto Acoplamiento

Métrica Sonar: Alto acoplamiento entre objetos (>10 dependencias)

Problema: Dependencia directa de implementaciones concretas

Violación (Código Malo):

public class OrderService {
    private MySQLOrderRepository repository = new MySQLOrderRepository();
    private SMSEmailService emailService = new SMSEmailService();
    
    public void processOrder(Order order) {
        repository.save(order);
        emailService.sendConfirmation(order);
    }
}
// Depende directamente de implementaciones concretas

Solución (Código Bueno):

public class OrderService {
    private final OrderRepository repository;
    private final NotificationService notifier;
    
    public OrderService(OrderRepository repo, NotificationService service) {
        this.repository = repo;
        this.notifier = service;
    }
    
    public void processOrder(Order order) {
        repository.save(order);
        notifier.sendConfirmation(order);
    }
}
// Depende de abstracciones, no de implementaciones
